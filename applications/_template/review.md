# Review Checklist

Use this file to review the application packet and capture improvements for future CVs, personal letters, answers, templates, or agent guidance.

## Packet QA

- [ ] `job.md` captures company, role, URL, language, requirements, questions, uploads, and constraints.
- [ ] `cv.html` uses only facts supported by `personal/`.
- [ ] `personal-letter.html` uses only facts supported by `personal/`, when a personal letter is needed.
- [ ] `application-answers.md` uses exact form questions and source-grounded answers.
- [ ] `evidence.md` lists missing or uncertain facts instead of letting them leak into drafts.
- [ ] Candidate-facing language matches the application/form language.
- [ ] Nothing has been typed into or submitted through the application form.

## CV Notes

- [ ] Summary feels specific to the role.
- [ ] Skills are ordered by relevance.
- [ ] Work bullets are relevant without changing fixed facts.
- [ ] Projects are relevant and not too many.
- [ ] Education/publication details are correct.
